以下のテーマでブログ記事を1本生成し、`content/blog/` にMarkdownファイルとして保存してください。

## 手順

### Step 1: 調査
- WebSearchツールでテーマに関する最新のトレンド、統計データ、事例を調査する

### Step 2: 画像の取得
- テーマに関連するフリー素材画像をUnsplashから取得する
- 以下のコマンドで `public/images/blog/` にダウンロードする:
  ```
  curl -sL -o public/images/blog/{ファイル名}.jpg "https://images.unsplash.com/photo-{ID}?w={幅}&q=80"
  ```
- 必要な画像:
  - **アイキャッチ画像**: 1枚（w=1200）
  - **挿入画像**: 2〜3枚（w=800）、各セクションに対応した画像
- 画像IDはUnsplashで検索して適切なものを選ぶ
- ダウンロード後、`file` コマンドで正常なJPEGであることを確認する
- 正常に取得できなかった場合はeyecatchを空にし、挿入画像なしで記事を生成する

### Step 3: 記事の執筆
- 調査結果をもとに、以下の記事構成に従って執筆する
- 画像は `/images/blog/{ファイル名}.jpg` のパスで参照する

### Step 4: 確認
- 生成した記事の全文をユーザーに表示し、「この内容で公開してよいですか？」と確認を求める

### Step 5: 公開
- ユーザーが承認したら、git add → git commit → git push origin main を実行してデプロイする
- ユーザーが修正を求めた場合は、指摘箇所を修正して再度確認を求める

## テーマ
$ARGUMENTS

テーマが指定されない場合は、建設DX・現場効率化・福祉DXに関連するトレンドトピックを自分で選んでください。

## 記事構成（この順番で書くこと）

### 1. リード文
- 読者の課題に共感する導入（2〜3文）

### 2. セクション1（現状・課題の解説）
- `##` 見出し
- 挿入画像を1枚配置: `![alt](/images/blog/{ファイル名}.jpg)`
- 業界の現状や課題を具体的に解説

### 3. セクション2（解決策・事例）
- `##` 見出し
- 挿入画像を1枚配置
- 具体的な解決策やツール、事例を紹介
- 外部サービスを紹介する場合は公式サイトへのリンクを貼る（ロゴ画像は使わない）

### 4. 中間CTA（自然な導線）
- セクション2の後に、記事の文脈に沿った自然な形でunionプロダクトへの導線を入れる
- 例: 「発注業務のデジタル化に興味がある方は、[unionの発注サービス](/services#service-scaffold)もご覧ください。」
- リンクはサイト内の相対パスを使う（`/services`, `/services#service-scaffold`, `/services#service-rental`, `/contact` など）。絶対URL（https://...）は使わない。
- 宣伝臭くならないよう、あくまで「関連情報」として提示する

### 5. セクション3（実践ステップ・ノウハウ）
- `##` 見出し
- 挿入画像を1枚配置（あれば）
- 読者がすぐ実践できるステップやポイントを提示

### 6. まとめ
- `##` 見出し
- 記事の要点を簡潔にまとめる

### 7. 記事末CTA
- まとめの後に以下のようなCTAセクションを入れる:
```markdown
---

**現場のDXを、小さく始めてみませんか？**

unionは建設・福祉の現場に特化したSaaSプロダクトを提供しています。発注、日報管理、勤怠管理など、現場の課題に合わせたツールをラインナップしています。

[unionのサービスを見る](/services) | [お問い合わせ](/contact)
```

## 会社情報（記事のトーン・文脈に使用）
- 会社名: 株式会社main character / ブランド名「union」
- 事業: 建設・福祉向け現場特化SaaSプロダクトの企画・開発・運営
- プロダクト: union 発注 for 足場、union 発注 for リース、union 健康管理、union 日報、union 勤怠管理
- ミッション: 現場の力を、テクノロジーでもっと前へ
- ターゲット: 建設業・福祉業界の現場で働くプロフェッショナル
- サイトURL: https://main-character.co.jp/

## カテゴリ（いずれかを選択）
- 業界トレンド（建設・福祉のDX、法改正、市場動向）
- 経営・マネジメント（中小企業の経営課題、補助金、DX投資判断）
- 業務改善（現場の一業務を切り取って効率化の方針を提示）
- プロダクト（unionの機能紹介・アップデート）
- ニュース（プレスリリース・お知らせ）
- 導入事例（顧客の成功事例）

## ファイル形式
- ファイル名: `{slug}.md`（英語のスラッグ、日付は不要）
- frontmatter:

```yaml
---
title: "記事タイトル"
category: "カテゴリ名"
publishedAt: "YYYY-MM-DDTHH:mm:ss+09:00"
eyecatch: "/images/blog/{アイキャッチファイル名}.jpg"
---
```

## 記事のルール
- 文字数: 2000〜3000字
- SEOを意識したタイトルと見出し構成
- 専門用語は分かりやすく説明
- 具体例や数値を含める
- 宣伝色は控えめに、読者に価値のある情報を提供
- 見出しは `##` から使用（`#` は使わない）
- 画像のalt属性は内容を適切に説明するテキストにする
- 外部サービスの紹介時はロゴを使わず、公式サイトへのリンクを貼る
